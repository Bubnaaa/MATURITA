## Principy, příkazy, skripty, Windows/Linux

---
### UI
- Způsob, jakým uživatel komunikuje s počítačem nebo programem
- Zprostředkovává vstupy mezi člověkem a systémem
- Dělí se na GUI a CLI
---
### GUI - Graphical User Interface
- Charakteristika
	- Grafické uživatelské rozhraní (okna, ikony, buttons)
	- Ovládání pomocí myši a klávesnice, nebo dotyku
	- Intuitivní, vhodné pro běžné uživatele
- Výhody:
	- Snadné a přehledné ovládání
	- Nižsí požadavky na znalosti uživatele
	- Vizualizace složitých procesů
	- Multitasking přes okna
- Nevýhody:
	- Vyšší zátěž na HW
	- Pomalejší pro pokročilé úlohy než CLI
	- Omezené možnosti automatizace
- GUI ve Windows
	- Pracovní plocha, nabídka Start, průzkumník souborů
	- Nastavení systému přes Ovládací panely / Nastavení 
	- Microsoft Store, notifikační centrum
- GUI v Linuxu
	- Desktopová prostředí: GNOME, KDE Plasma, XFCE
	- Správce oken, panel, terminál integrovaný přímo do GUI
	- Další balíčkové manažery pro vlastní GUI

---
### CLI - Command Line Interface
- Charakteristika
	- Textové rozhraní ovládané příkazy
	- Uživatel zadává instrukce pomocí klávesnice
	- Win CMD, PowerShell, Linux BASH
- Výhody:
	- Rychlost, přesnost
	- Nízké nároky na paměť a výkon
	- Možnost psát scripty -> automatizace
	- Lepší kontrola nad systémem, ideální pro adminy
- Nevýhody:
	- Nároky na znalosti příkazů
	- Není vizuální
	- Přesná syntaxe
---
### Windows CMD
- `dir` – výpis souborů
- `cd` – změna adresáře
- `copy`, `move`, `del` – manipulace se soubory
- `ipconfig` – síťová konfigurace
- `tasklist`, `taskkill` – procesy
- `ping`, `tracert` – síťové testy
### Základní příkazy – Linux (bash) 

- `ls` – výpis souborů
- `cd`, `pwd`
- `cp`, `mv`, `rm`, `mkdir`
- `cat`, `less`, `nano`, `vim`
- `apt install`, `apt update` (Debian/Ubuntu)
- `chmod`, `chown` – práva a vlastnictví
- `ps`, `top`, `kill` – procesy
- `ping`, `traceroute`, `ifconfig/ip` – síťové nástroje
